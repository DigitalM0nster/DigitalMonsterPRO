# Позы About → Blender → обратно на сайт

Нужно: открыть текущие камеру и модель из сайта в Blender, покрутить их глазами, потом вернуть цифры на сайт.

---

## Что получится

В Blender появятся три «куклы»:

- **AboutCamera** — камера как на сайте  
- **AboutLookAt** — точка, куда камера смотрит  
- **AboutModel** — пустышка с кейфреймами позы  
- **AboutModelAsset** — сюда вешаешь GLB/меш (как на сайте)

На таймлайне 5 кадров = 5 этапов story:

| Этап на сайте | Кадр в Blender |
|---------------|----------------|
| story 0       | кадр **1**     |
| story 1       | кадр **11**    |
| story 2       | кадр **21**    |
| story 3       | кадр **31**    |
| story 4       | кадр **41**    |

Крутишь таймлайн → видишь позу этапа → двигаешь объекты → экспортируешь.

---

## Часть 1. Один раз настроить сцену

### 1. Открой Blender

Любая версия 3 или 4. Можно пустой файл (File → New → General).

### 2. (По желанию) закинь модель

Чтобы видеть ту же железку, что на сайте:

1. `File` → `Import` → `glTF 2.0`  
2. Выбери файл сайта:  
   `public/models/aboutModel/AboutUsModel.glb`  
3. Пока модель просто лежит в сцене — привяжем её на шаге 6.

Если модели нет под рукой — можно править только пустышки и камеру, без GLB.

На сайте модель **не масштабируется** (`ABOUT_MODEL_TARGET_SIZE = 0`) — тот же размер, что GLB в Blender. Иначе камера на тех же метрах будет «ближе».

### 3. Открой скрипт

1. Вверху переключи workspace на **Scripting** (или окно Text Editor).  
2. `Open` → файл:  
   `tools/about-blender/about_stage_poses_blender.py`  
3. В начале файла должно быть:

```python
MODE = "import"
FORCE_RESET = False
```

**Осторожно:** `import` пересоздаёт пустышки и **стирает** кейфреймы `AboutModel` / камеры.  
Делай это один раз в начале. Потом всегда `MODE = "export"`.  
Повторный сброс только с `FORCE_RESET = True`.

### 4. Запусти импорт

Нажми ▶ **Run Script** (или Alt+P).

Внизу в System Console / Info должно появиться что-то вроде:  
`[AboutPoses] Чистая сцена...`

### 5. Найди объекты

Слева в Outliner (список объектов) ищи коллекцию **AboutStagePoses**:

- `AboutRoot`  
- `AboutModel` — кейфреймы поз  
- `AboutModelAsset` — сюда parent’ь меш/GLB  
- `AboutCamera`  
- `AboutLookAt`

Кликни `AboutCamera`, нажми **Numpad 0** — смотришь из этой камеры (как на сайте).

### 6. Привяжи GLB к AboutModelAsset (если импортировал модель)

1. В Outliner выбери свой импортированный объект модели (часто называется `AboutUsModel` или Scene).  
2. Затем **с Shift** выбери `AboutModelAsset` (она должна быть активной — ярче).  
3. `Ctrl + P` → **Object** (parent).  

Не parent’ь к `AboutModel` — только к `AboutModelAsset`.

---

## Часть 2. Правишь позы

1. Переключись обратно в workspace **Layout** (удобнее смотреть 3D).  
2. Внизу таймлайн поставь кадр **1** (story 0).  
3. Выбери `AboutModel` или `AboutCamera` / `AboutLookAt`.  
4. Двигай:  
   - **G** — переместить  
   - **R** — крутить  
   - **S** — масштаб не трогай (на сайт не уходит)  
5. Обязательно зафиксируй кейфрейм: выдели объект → **I** → Location & Rotation  
   (иначе при смене кадра правка пропадёт).  
6. Перейди на кадр **11**, **21**, **31**, **41** — то же самое для других этапов.  
   Ключи можно ставить и на **промежуточные** кадры — экспорт сам найдёт их в сегменте story.

Камера на сайте смотрит в `AboutLookAt`. Если двигаешь только камеру, а цель стоит — ракурс меняется. Часто удобнее двигать **LookAt** или **Model**.

Экспорт печатает `Keys camera / lookAt / model` и кадр, с которого снята каждая story-поза.

---

## Часть 3. Вернуть на сайт

### 1. Снова открой скрипт

В Text Editor в том же файле поменяй строку:

```python
MODE = "export"
```

### 2. Run Script

В консоли появится большой блок:

```js
export const ABOUT_STAGE_POSES = [
  // story 0
  { ... },
  ...
];
```

Также появятся файлы рядом со скриптом:

- `ABOUT_STAGE_POSES.export.js`  
- `aboutStagePoses.export.json`

### 3. Что делать с результатом

**Вариант А.** Скопируй весь `export const ABOUT_STAGE_POSES = [...]` и пришли мне в чат — я вставлю в проект.

**Вариант Б.** Сам вставь содержимое в  
`src/three/scenes/about/aboutStagePoses.js`  
внутрь функции `createDefaultAboutStagePoses()` (замени старый массив).

Потом на сайте (dev): клавиша **4** → Reset, если панель ещё держала старые значения в памяти, или просто обнови страницу.

## Если что-то пошло не так

| Проблема | Что сделать |
|----------|-------------|
| Скрипт ругается на JSON | Рядом со `.py` должен лежать `aboutStagePoses.json` (он уже в папке). |
| Не вижу объектов | Outliner → ищи коллекцию `AboutStagePoses`. Запусти import ещё раз. |
| Модель не едет / перевёрнута | Parent должен быть **AboutModelAsset**. На сайте убран старый +90° по X. |
| Правки сбрасываются | Не забыл **I** (Insert Keyframe) на нужном кадре. |
| Не знаю, куда смотрит камера | Выдели `AboutCamera` → Numpad **0**. |

---

## Коротко

1. `MODE = "import"` → Run  
2. Parent GLB → `AboutModelAsset`  
3. Кадры 1 / 11 / 21 / 31 / 41 → крути → **I**  
4. `MODE = "export"` → Run  
5. Пришли JS сюда или вставь в `aboutStagePoses.js`
