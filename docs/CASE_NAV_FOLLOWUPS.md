# Case navigation — доработки

Зафиксировано при смене модели навигации кейса (дуга = список проектов, стадии — слева).

## Дуга: циклическое кольцо проектов

**Сейчас:** все проекты реестра стоят на циклическом кольце (`gap` между соседями). Активный всегда уезжает в центр дуги кратчайшим путём; соседи заворачиваются (01 ↔ последний). Вне клина дуги узлы гаснут по fade.

**Доработать при необходимости:**

1. Если проектов станет очень много — уплотнять `gap` или показывать только ближайшее окно `maxNavItems` с тем же cyclic wrap.
2. Клики и glow остаются дешёвыми (без Canvas2D на каждый кадр скролла списка).

Критерий: все проекты портфолио доступны с дуги; текущий кейс всегда в центре активной зоны.

## Уже сделано в этой итерации

- Дуга: названия проектов (не стадии кейса).
- Убраны нижние «предыдущий / следующий проект».
- «Все проекты» — компактно над фиксированным заголовком кейса.
- Стадии кейса — рельса слева от текстового блока.
- Бейдж `01 / …` + footer проекта закреплены на одной `contentTopPx` во всех кейсах.
